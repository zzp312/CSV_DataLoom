## Context

本项目旨在构建一款面向数据分析师和开发者的轻量级桌面端ETL工具——"本地数据双模工作台"。核心挑战在于处理十万级以上数据时的性能瓶颈，以及提供兼顾小白用户（可视化拖拽）和专业开发者（手写SQL）的双模操作体验。

现有方案中，基于Pandas的全内存加载方式在大数据量场景下会导致内存溢出和界面卡顿。因此需要引入嵌入式列式数据库DuckDB来实现外存查询和向量化执行，同时采用分页加载技术保证界面流畅性。

## Goals / Non-Goals

**Goals:**
- 支持CSV/XLSX文件导入，多Sheet选择与表名重命名
- 实现可视化画布模式：拖拽建表、连线关联、字段勾选、筛选条件配置
- 实现SQL编辑器模式：语法高亮、自动补全、双向同步
- 支持DuckDB引擎的高性能数据处理，百万行数据查询秒级响应
- 分页加载技术，界面仅渲染可见数据，保证流畅体验
- 一键导出CSV和智能SQL脚本生成
- 跨平台支持（Windows/Mac/Linux）

**Non-Goals:**
- 不支持远程数据库连接（仅本地文件处理）
- 不提供数据可视化图表功能（专注于ETL）
- 不支持实时数据同步
- 不提供用户认证和权限管理

## Decisions

### 1. GUI框架选择：Flet

**选择理由：**
- 基于Google Flutter引擎，具备Material Design风格，UI美观
- 纯Python开发，学习成本低，开发效率高
- 跨平台打包方便（Windows/Mac/Linux）
- 提供PaginatedDataTable组件，天然支持分页加载
- 支持拖拽功能（Draggable/DragTarget）

**替代方案对比：**
- PyQt：功能强大但学习曲线陡峭，UI风格较传统
- Tkinter：轻量但UI美观度不足
- Electron：跨平台但体积大，启动慢

### 2. 数据处理引擎：DuckDB

**选择理由：**
- 嵌入式列式数据库，零配置启动
- 支持向量化执行，GROUP BY和JOIN操作比Pandas快数倍
- 支持直接对CSV/Parquet文件进行SQL查询，无需全量加载内存
- 完全兼容SQL标准语法
- 支持Python API，集成简单

**替代方案对比：**
- Pandas：全内存加载，大数据量场景内存溢出
- SQLite：行式存储，分析查询性能不如列式数据库
- Polars：性能优秀但生态不如DuckDB成熟

### 3. 界面布局：左-中-右三栏式

**设计：**
- 左侧：数据资产导航栏（表结构管理）
- 中间：可视化画布与SQL编辑器（双模核心）
- 右侧：结果预览与导出控制台

**理由：**
- 经典的IDE布局，符合用户习惯
- 各区域职责清晰，操作流畅
- 支持双模操作无缝切换

### 4. 分页加载策略

**实现方案：**
- 使用Flet PaginatedDataTable组件
- 默认每页显示50-100行
- 通过SQL LIMIT和OFFSET实现动态分页查询
- 底部显示总行数和查询耗时

**理由：**
- 避免一次性渲染大量数据导致界面卡顿
- 降低内存占用，提升响应速度
- 支持快速跳页和滚动加载

### 5. SQL解析与双向同步：SQLGlot

**选择理由：**
- 支持多种SQL方言解析和转换
- 可以将画布操作转化为SQL语句
- 可以解析SQL语句反向更新画布状态
- 支持SQL格式化和语法验证

## Risks / Trade-offs

| Risk | Mitigation |
|------|------------|
| DuckDB学习曲线 | 提供详细的API文档和示例代码，封装通用查询方法 |
| 复杂SQL反向解析失败 | 当解析失败时提示"复杂查询仅支持代码模式"，保留SQL编辑器功能 |
| 大数据量导出性能 | 分批导出，避免一次性加载全部数据到内存 |
| 跨平台兼容性 | 使用Flet官方打包工具，测试各平台兼容性 |
| Excel文件解析兼容性 | 使用Pandas作为辅助库处理Excel读取，支持多种格式 |
